# weighted-pca-codes

This repository contains the code and data needed to replicate the numerical results reported in the revised manuscript on weighted PCA.

The repository is organized around the paper outputs. Each figure or table has its own entry-point script in `scripts/`, so results can be reproduced one by one without using a single monolithic runner.

No pre-generated figures or tables are included in the repository. When a script is run, it creates an `outputs/` folder locally and writes the corresponding files there.

## Repository Structure

- `scripts/`
  - one script per paper figure or table
- `R/`
  - shared helper functions used by the entry-point scripts
- `original/codes/`
  - preserved copies of the original core code and required data/cache files
- `original/revision_figure2/`
  - preserved revision script used for Figure 2
- `original/revision_figure7/`
  - preserved revision script used for Figure 7

## How To Run

Run scripts individually from the repository root. For example:

```r
Rscript scripts/Figure2.R
Rscript scripts/Table2.R
Rscript scripts/Table5.R
```

This repository intentionally does not rely on a `run_all` script. The main replication workflow is to run the individual script corresponding to the figure or table of interest.

## Optional Arguments

- `Figure1.R`, `Figure3.R`, `Figure4.R`, `Figure5.R`, `Figure6.R`
  - first argument: `n_rep`
  - second argument: `seed`

- `Figure2.R`
  - first argument: `n_rep`
  - second argument: `seed`

- `Figure7.R`
  - first argument: `seed`

- `Table2.R`
  - first argument: `n_rep`
  - second argument: `seed`

- `Table3.R`, `Table4.R`, `Table5.R`
  - default mode is `cached`
  - use `recompute` as the first argument to rerun from scratch

Examples:

```r
Rscript scripts/Figure2.R 300 20260315
Rscript scripts/Table3.R cached
Rscript scripts/Table5.R recompute 100 1234
```

## Notes

- The repository is designed so that each paper result can be reproduced from a clearly named script.
- The generated `outputs/` folder is not part of the repository snapshot and is created only when scripts are run locally.
- For Figure 2, small test runs can produce a visibly noisier trend. The manuscript-style default is the no-argument run, which uses `n_rep = 300` and `seed = 20260315`.
